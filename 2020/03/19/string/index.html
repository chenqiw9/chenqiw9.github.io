一道非常有意思的题目，当初困惑了我不久。

64位，开了这么多有点麻烦，运行一下程序 ……….额程序有点长，做的选择比较多，直接去IDA看吧

看了蛮久的代码，终于看懂了大概过程，首先输入名字，然后输入要去的方向（只能去east）然后进行左右选择（只能选1）接着他让我们输入一个地址，后面是输入你的愿望，把你的愿望打印两遍，这里可以发现格式化字符串漏洞，看到最后一个函数，只要满足a1=a1[1]就可以进行一次read输入，不过这里我们遇到了mmap函数，百度一下可以发现mmap将一个文件或者其它对象映射进内存

刚好我们写入的数据就是到了v1中，这就说明我们可以直接在这写入shellcode便可以直接执行 （有关于shellcode） 这里我们可以调用pwntools自带的,asm(shellcraft.sh() )

那么现在的问题就是怎么让a1=a1[1],在这个之前我们发现了一个格式化字符串漏洞，因此我们可以利用一下这个漏洞，使得a1=a1[1] ，接下来就要计算偏移

数了一下，偏移为8，但是因为在64位程序中，如果我们将地址打包会生成好多个‘00’，而在字符串中‘00’就代表结束，所以在printf到‘00’时，就被认为字符串已经结束了，自然不会继续往后面printf了，也即是说我们的字符串都被’00’给截断了，所以我们可以把地址放在后面，那么偏移就是九了。 a1的值是68，a2的值是85。

在这里他将地址打印了出来，这里我们使用addr0 = int(p.recvuntil(‘\n’),16)将接受到的数据保存在addr0中
